        <h2>Docs</h2>
        <p>How to use?</p>
        <aside>
            <p>If you know how to add a picture in markdown, then you are good to go.</p>
        </aside>
        <p>All you have to do is to add the following <mark>markdown markup</mark> to your markdown content:</p>
        <pre>
            <code lang="markdown">
                ![visitors](https://visitor-badge.laobi.icu/badge?page_id=page.id)
            </code>
        </pre>
        <p>In which, the URL parameter <mark>page_id</mark> is <b>REQUIRED</b>, please use a unique string to best represent your page.</p>
        <p id="page_id_rule">I recommend you to follow the <code>page_id</code> rules below:</p>
        <ol>
            <li>For README.md file, use <i>${your.username}.${your.repo.id}</i>, <code>https://visitor-badge.laobi.icu/badge?page_id=jwenjian.visitor-badge</code> for example.</li>
            <li>For Issue body, use <i>${your.username}.${your.repo.id}.issue.${issue.id}</i>, <code>https://visitor-badge.laobi.icu/badge?page_id=jwenjian.visitor-badge.issue.1</code> for example.</li>
        </ol>
        <p>or any other markdown content, please give a unique string to distinguish</p>
        <p>The URL parameter <mark>left_color</mark> and <mark>right_color</mark> are optional; you can customize the badge color by passing the two parameters.</p>
        <p>The values support color names (like green) and RGB in hex string that starts with a hashtag (like #66ccff)</p>
